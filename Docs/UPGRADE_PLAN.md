# 🚀 《僵尸防线》升级策划文档

## 一、核心玩法创新点

### 1.1 「塔防+背包管理」融合玩法

#### 设计概念
将背包空间管理与塔防策略结合，玩家需要在有限的背包空间内配置防御塔、道具和增益效果。

#### 具体机制
```
背包系统：
- 9宫格/16宫格背包空间
- 不同防御塔占用不同格子数（1格/2格/4格/L型）
- 同类型相邻可合成升级
- 背包扩展通过局外养成解锁

战斗机制：
- 每波开始前30秒整理背包
- 战斗中可拖拽调整塔位置（有限次数）
- 紧急情况下可"丢弃"塔腾出空间
```

#### 创新价值
- 增加策略维度：空间规划+塔防布局
- 提升单局变数：背包随机刷新塔
- 差异化竞争：与《向僵尸开炮》形成差异

### 1.2 「元素反应链」系统

#### 设计概念
借鉴《原神》元素反应机制，不同属性防御塔组合触发连锁反应。

#### 元素类型
| 元素 | 基础效果 | 组合反应 |
|------|----------|----------|
| 火 | 持续灼烧 | 火+冰=蒸发（伤害×2） |
| 冰 | 减速冻结 | 火+电=超载（范围爆炸） |
| 电 | 连锁弹射 | 冰+电=超导（破甲效果） |
| 毒 | 持续伤害 | 毒+火=燃烧（扩散伤害） |
| 风 | 击退聚拢 | 风+任意=扩散（范围传播） |

#### 技能设计示例
```
【火焰塔】
- 基础：发射火球造成100%伤害，附加灼烧
- 连携1（冰）：触发蒸发，伤害200%
- 连携2（电）：触发超载，范围150%伤害

【冰霜塔】
- 基础：发射冰箭造成80%伤害，减速30%
- 连携1（火）：触发蒸发，伤害200%
- 连携2（电）：触发超导，无视50%防御
```

### 1.3 「动态地形」机制

#### 设计概念
战场地形随波次变化，玩家需要动态调整策略。

#### 地形类型
| 地形 | 效果 | 应对策略 |
|------|------|----------|
| 熔岩地带 | 经过敌人持续掉血 | 引导敌人绕行 |
| 冰冻地面 | 敌人减速50% | 布置高伤塔 |
| 高地 | 塔射程+30% | 优先占领 |
| 障碍物 | 阻挡直线攻击 | 布置曲射塔 |
| 传送门 | 敌人瞬移 | 多点布防 |

#### 动态事件
```
第5波：随机生成3个熔岩地带
第10波：地图中央出现高地
第15波：两侧出现传送门
BOSS波：地形完全重组
```

### 1.4 「英雄单位」系统

#### 设计概念
引入可移动操控的英雄单位，增加操作维度。

#### 英雄类型
| 英雄 | 定位 | 技能 |
|------|------|------|
| 重装战士 | 坦克 | 嘲讽、护盾、反击 |
| 狙击手 | 输出 | 爆头、穿透、标记 |
| 工程师 | 辅助 | 修塔、加速、召唤 |
| 法师 | 群攻 | 陨石、冰冻、传送 |

#### 操作方式
- 点击地图移动
- 自动攻击范围内敌人
- 手动释放技能（CD制）
- 可携带装备和符文

## 二、新增系统功能

### 2.1 深度养成系统

#### 2.1.1 装备系统
```
装备部位：
- 武器：攻击力、暴击率
- 护甲：生命值、减伤
- 饰品：特殊效果（吸血、反弹等）

品质等级：
白→绿→蓝→紫→橙→红→彩

装备获取：
- 关卡掉落
- 合成升级（5合1）
- 商店购买
- 活动奖励

洗练系统：
- 消耗材料重置词条
- 锁定优质词条
- 追求极限属性
```

#### 2.1.2 芯片/符文系统
```
芯片类型：
- 攻击芯片：加攻击、暴击、穿透
- 防御芯片：加生命、减伤、回复
- 功能芯片：加金币、经验、掉落

镶嵌规则：
- 每个装备5个芯片槽
- 同类型芯片有套装效果
- 芯片可升级、可拆卸
```

#### 2.1.3 天赋/科技树
```
天赋分支：
├─ 攻击系：伤害加成、暴击强化、元素精通
├─ 防御系：生命加成、减伤、回复强化
├─ 经济系：金币加成、成本降低、利息系统
└─ 特殊系：额外技能槽、背包扩展、复活次数

重置机制：
- 首次免费重置
- 后续消耗钻石
- 支持多套天赋方案
```

## 三、美术风格定位

### 3.1 风格选择：赛博朋克 × 塔防

#### 为什么选择赛博朋克？
| 优势 | 说明 |
|------|------|
| 差异化 | 避开泛滥的丧尸/修仙题材 |
| 视觉冲击力 | 霓虹灯、全息投影、机械元素 |
| 目标用户 | 契合18-35岁核心用户审美 |
| 延展性 | 易于推出皮肤、赛季主题 |

#### 视觉元素
```
色彩方案：
- 主色调：深紫、电蓝、霓虹粉
- 辅助色：黑色、银色、荧光绿
- 强调色：橙色（警告）、红色（危险）

场景设计：
- 高楼林立的未来都市
- 全息广告牌和霓虹灯
- 数据流和电路板纹理
- 雨夜和雾气效果

防御塔设计：
- 机械结构外露
- LED灯光效果
- 能量护盾和力场
- 变形动画

敌人设计：
- 机械僵尸：破损的机器人
- 黑客程序：数据流构成的幽灵
- 企业安保：全副武装的士兵
- 赛博精神病：失控的改造人
```

### 3.2 UI设计规范

#### 主界面
```
布局：
- 中央：3D场景展示（可旋转）
- 底部：功能按钮（开始游戏、养成、商店等）
- 顶部：资源显示（金币、钻石、体力）
- 右侧：活动入口

动效：
- 按钮悬停：霓虹灯闪烁效果
- 转场：数据流穿梭效果
- 提示：全息投影弹出
```

#### 战斗界面
```
布局：
- 顶部：波次、金币、生命值
- 底部：技能栏、背包栏
- 左侧：塔选择面板
- 右侧：加速/暂停按钮

特效：
- 伤害数字：像素风格+霓虹描边
- 技能特效：粒子效果+光晕
- 受击反馈：屏幕震动+红色闪烁
```
